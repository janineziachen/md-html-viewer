# Markdown 编辑与高亮提取功能 — 设计文档

日期：2026-06-04
状态：已与用户确认，待转入实现计划

## 1. 背景与目标

MobileMD 目前对 Markdown 只做只读渲染。用户希望在阅读 Markdown 时能够编辑：
修改原文、加粗、高亮重点、并把所有高亮内容统一提取出来用于复习/回顾。

用途保持中性——用户高亮可能是重点、待办、喜欢的句子等，不绑定「考试」语义，
因此所有面向用户的措辞使用「高亮」「高亮提取」，不使用「考点」。

## 2. 范围

### 做
- Markdown「阅读 / 编辑」模式切换（仅 Markdown 格式）。
- 编辑模式：源码 `<textarea>` + 格式工具栏（加粗、高亮）+ 使用引导文案。
- 加粗 `**x**`、高亮 `==x==` 两种标记；正文可任意增删改。
- 高亮渲染：`==x==` 渲染成 `<mark>`（黄色底，深色模式适配）。
- 保存：弹出选择「覆盖原条 / 另存为新条」；另存与导出均让用户自定标题。
- 阅读模式「只看高亮」开关：仅显示所有高亮句子，可随时切回全文。
- 「导出高亮」：抽取全部高亮内容，存成一条独立历史记录。

### 不做（本期）
- 字号调整（Markdown 无标准语法，暂不做）。
- 所见即所得（WYSIWYG）编辑。
- 颜色修改。
- 专门的笔记按钮（记笔记 = 直接在源码打字）。
- 导出为 `.md` 文件下载（保存只写回 IndexedDB 历史）。

## 3. 关键决策（用户已确认）

| 决策点 | 选择 |
| --- | --- |
| 编辑方式 | 源码编辑 + 格式工具栏 |
| 字号 | 暂不做 |
| 高亮语法 | `==x==`，用 remark 高亮插件渲染成 `<mark>` |
| 笔记 | 不单设按钮，直接改文字 |
| 保存 | 保存时询问：覆盖原条 / 另存为新条 |
| 另存 / 导出标题 | 让用户自定（预填默认值，可改） |
| 高亮提取 | 两者都要：页内「只看高亮」开关 + 「导出高亮」生成独立历史 |
| 措辞 | 中性「高亮」，不用「考点」 |

## 4. 架构与数据流

### 4.1 组件结构

`PreviewScreen`（Markdown 分支）新增「阅读 / 编辑」模式切换，仅对 `format === 'markdown'` 显示。

```
PreviewScreen (markdown)
├── 模式切换：阅读 | 编辑
├── 阅读模式
│   ├── 「只看高亮」开关
│   ├── 「导出高亮」按钮
│   └── MarkdownRenderer（已支持 ==x== → <mark>）
│       └── 只看高亮时：仅渲染抽取出的高亮句子列表
└── 编辑模式
    ├── 格式工具栏：[B 加粗] [高亮]
    ├── 引导文案：直接打字改原文；选中文字后点上方按钮加粗或高亮
    ├── <textarea>（源码，本地 draft 状态）
    └── 「保存」按钮 → 弹出：覆盖原条 / 另存为新条
```

### 4.2 数据流

- `App` 的 `current` 增加 `id` 字段，把历史记录 id 透传给 `PreviewScreen`，保存时才知道覆盖哪条。
- 编辑模式持有本地 `draft` 字符串；点「保存」时回调 `onSave`。
- 保存分支：
  - **覆盖原条** → `updateHistory(id, draft)`，同时更新 `current.content`。
  - **另存为新条** → `addHistory({ format:'markdown', content: draft, title })`，标题用户自定，预填 `原标题（已编辑）`。
- 「导出高亮」→ `extractHighlights(content)` 拼成新 Markdown → `addHistory(...)`，标题预填 `原标题 · 高亮`，可改。

### 4.3 高亮渲染

remark 高亮插件把 `==x==` 转成 `<mark>`。`<mark>` 样式：浅色模式黄色底，深色模式降低亮度的暖色底，保证可读。

### 4.4 关键函数与改动点

| 位置 | 改动 |
| --- | --- |
| `src/lib/highlight.ts`（新） | `extractHighlights(md: string): string[]` 纯函数，按出现顺序抽取所有 `==...==` 内容；供「只看高亮」与「导出高亮」共用 |
| `src/lib/history.ts` | 新增 `updateHistory(id, content)` |
| `src/components/renderers/MarkdownRenderer.tsx` | 接入 remark 高亮插件 |
| `src/components/PreviewScreen.tsx` | 阅读/编辑模式切换、工具栏、保存弹窗、只看高亮、导出高亮 |
| `src/App.tsx` | `current` 增加 `id`，透传并处理 `onSave` |
| `src/types.ts` | 无需改 `HistoryItem`；`current` 内部类型加 `id` |

### 4.5 编辑引导（教用户怎么用）

- 工具栏按钮带文字标签：`B 加粗`、`高亮`。
- 引导行：`直接打字改原文；选中文字后点上方按钮加粗或高亮`。
- 选中文字后点按钮 → 包裹 `**...**` / `==...==`。
- 未选中就点按钮 → 插入占位符 `**加粗文字**` / `==高亮文字==`，并自动选中占位词，方便直接替换。

## 5. 错误处理

- 非 Markdown 格式：不显示编辑入口（沿用只读渲染）。
- 保存时 draft 为空：禁用「保存」。
- 另存 / 导出标题为空：回退到预填默认值。
- IndexedDB 写入失败：toast/提示「保存失败，请重试」，不丢弃 draft。

## 6. 测试策略

- `extractHighlights` 单元测试：多个高亮、跨行、无高亮、`==` 不成对等边界。
- 高亮渲染测试：`==x==` 渲染出 `<mark>`。
- 工具栏插入测试：选中包裹 / 未选中插占位符。
- 保存路径测试：覆盖原条调用 `updateHistory`、另存调用 `addHistory`。
- 只看高亮测试：仅显示高亮句子，可切回全文。
